Computer learning path

计算机知识结构组成

学科知识:

数据结构 计算机网络 操作系统 计算机组成原理

编译原理 数学 基础理论

编程技能:

编程语言 数据库 编译器 设计模式 软件工程…

方向分类

研发方向:

人工智能 体系结构 数据方向 算法设计 系统研发 …

开发方向:

后端开发 前端开发 游戏开发 嵌入式开发 移动开发 …

交叉方向:

生物信息 医学信息 计算化学 教育技术 计算物理学 …

研发方向:

学习现有技术,研究新的技术突破和解决方案

核心掌握:数学 计算机学科知识 技术底层

开发方向:

学习现有技术,运用已有的技术实现需求和解决问题,强调实践能力

掌握一门编程语言和相关技术体系,有项目

算法能力和计算机学科基础掌握情况

竞赛(ACM PAT 蓝桥国赛) 实习

公司体量越大,越注重顶层设计和底层优化能力,注重数据结构 算法 操作系统 网络…

开发的核心是实现需求和解决问题的能力,能组织多少人实现多大的需求就是我们的最终成长方向

关于编程语言

编程语言划分: 开发主体语言 开发辅助语言

(入门后,深入C++或Java,辅修一门语言)

编程语言的核心区别在于特性和涉及的领域不同

读研方向

大数据 人工智能 软工 计科 硬件软件…

数据分析 数据挖掘

导师选择:

师兄师姐去向是重要参考

走导师方向还是进大厂开发

走导师方向:梭哈跟导师安排,多听师兄师姐建议

走大厂开发:多学开发技能,多刷题,尽早实习(研二开冲)

师兄师姐考公或者转行 : 准备提桶跑路,多做项目积累履历

硕士重点:

研一: 完成日常上课和导师任务,积累足够的就业技能(开发:深入一门编程语言和开发相关技术体系, 算法:机器学习, 深度学习相关框架和模型,看大量论文),保持长期刷题习惯

研二: 多做项目实践,关注实习招聘信息 针对训练,背面经 准备简历

(Github Kaggle上能找到很多项目,善用开源平台实践)

少jb考乱七八糟的证书,多刷题 多准备项目比什么都强

不要怀疑自己是不是笨 是不是不适合计算机学习, 冲冲冲

# 学习侧重

考研复试: 项目 && 竞赛

猛冲四大件

优先学黑皮书,不需要过早接触考研教材

先听课建立基本了解,再通过书籍深入,最后项目实践

数据结构(acwing + codeforces + leetcode) -> (没错后面这俩一起学)计算机组成原理(CSAPP + 计算机组成与设计) + 操作系统(清楚系统资源调配,学会工程解决思路 CSAPP + 操作系统导论) -> 计算机网络(计算机网络:自顶向下方法)
